# Internal Boot FAQ

This page contains frequently asked questions about internal boot in Unraid OS. For general licensing questions, see the [Licensing FAQ](../../troubleshooting/licensing-faq.mdx). For TPM-based licensing questions, see the [TPM Licensing FAQ](../../troubleshooting/tpm-licensing-faq.mdx).

---

### Does an internal boot SSD or NVMe device count toward my total attached storage device limit? {#internal-boot-drive-count}

Yes. Internal boot SSD or NVMe devices count toward your total attached storage device limit.

Unraid does not currently exclude internal boot drives from the used device count. If you use internal boot, those drive(s) take up license slots. If you use mirrored internal boot, both boot devices count.

### Can I configure internal boot with two SSDs in a mirrored setup for redundancy? {#internal-boot-mirror}

Yes. Mirrored internal boot is supported by default.

Internal boot uses %%ZFS|zfs%%, and mirrored internal boot uses a %%ZFS mirror|zfs%% for redundancy. If one boot drive fails, Unraid continues to function and the failed device appears in the OS as degraded. You can then replace the failed drive using the normal Unraid drive assignment flow to restore the mirror.

### What should I check if I switch to internal boot and the system still boots from flash? {#internal-boot-bios-order}

If the system continues to boot from flash after switching to internal boot, manually change the boot order in your BIOS.

### Can I enable internal boot and still license from flash? {#internal-boot-flash-license}

Yes, the licensing method is not tied to the boot method.

For TPM-based licensing details, see the [TPM Licensing FAQ](../../troubleshooting/tpm-licensing-faq.mdx).

### What should I do if I see "prohibited by secure boot policy" after enabling internal boot? {#internal-boot-secure-boot}

Disable **Secure Boot** in your BIOS.
