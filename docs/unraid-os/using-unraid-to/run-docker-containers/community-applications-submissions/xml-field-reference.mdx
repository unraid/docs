# XML Field Reference

The submission parser supports the public XML fields below. Fields marked as minimum are expected for the basic submission path. Recommended fields improve review quality or the listing presentation. Advanced fields are accepted by the parser but are more specialized or legacy-oriented.

## Docker template fields

| Tag             | Level       | Notes                                                                             |
| --------------- | ----------- | --------------------------------------------------------------------------------- |
| `<Name>`        | Minimum     | Display name shown in Community Applications.                                     |
| `<Repository>`  | Minimum     | Docker image reference for the application.                                       |
| `<Category>`    | Recommended | Category string used for app organization.                                        |
| `<Icon>`        | Recommended | Icon URL for the app or maintainer profile.                                       |
| `<Overview>`    | Recommended | Primary summary shown in Community Applications.                                  |
| `<Project>`     | Recommended | Project home page or source repository.                                           |
| `<Support>`     | Recommended | Support URL for forums, issues, docs, or community help.                          |
| `<TemplateURL>` | Recommended | Canonical raw template URL used for identity and review.                          |
| `<Beta>`        | Advanced    | Marks the application as beta when set to `true`.                                 |
| `<Config>`      | Advanced    | Structured configuration entries for paths, ports, variables, devices, or labels. |
| `<Deprecated>`  | Advanced    | Marks the application as deprecated.                                              |
| `<Description>` | Advanced    | Longer description text. Use `<Overview>` for the primary summary.                |
| `<DonateLink>`  | Advanced    | Donation or sponsorship URL.                                                      |
| `<DonateText>`  | Advanced    | Call-to-action text paired with `<DonateLink>`.                                   |
| `<ExtraParams>` | Advanced    | Additional Docker run arguments. Unsafe values can be rejected.                   |
| `<MyIP>`        | Advanced    | Legacy networking placeholder.                                                    |
| `<Network>`     | Advanced    | Docker network mode.                                                              |
| `<Privileged>`  | Advanced    | Privileged container flag.                                                        |
| `<ReadMe>`      | Advanced    | Optional readme URL.                                                              |
| `<Registry>`    | Advanced    | Registry home page URL.                                                           |
| `<Requires>`    | Advanced    | Requirements text.                                                                |
| `<Shell>`       | Advanced    | Shell used for container exec actions.                                            |
| `<WebUI>`       | Advanced    | Web UI launch URL template.                                                       |

## Plugin template fields

| Tag            | Level       | Notes                                                    |
| -------------- | ----------- | -------------------------------------------------------- |
| `<PluginURL>`  | Minimum     | URL to the Unraid plugin manifest (`.plg`).              |
| `<Name>`       | Minimum     | Display name shown in Community Applications.            |
| `<Category>`   | Recommended | Category string used for app organization.               |
| `<Icon>`       | Recommended | Icon URL for the app or maintainer profile.              |
| `<Overview>`   | Recommended | Primary summary shown in Community Applications.         |
| `<Project>`    | Recommended | Project home page or source repository.                  |
| `<Support>`    | Recommended | Support URL for forums, issues, docs, or community help. |
| `<Beta>`       | Advanced    | Marks the plugin as beta when set to `true`.             |
| `<Deprecated>` | Advanced    | Marks the plugin as deprecated.                          |
| `<DonateLink>` | Advanced    | Donation or sponsorship URL.                             |
| `<DonateText>` | Advanced    | Call-to-action text paired with `<DonateLink>`.          |
| `<ReadMe>`     | Advanced    | Optional readme URL.                                     |

## Repository profile fields

| Tag            | Level       | Notes                                                |
| -------------- | ----------- | ---------------------------------------------------- |
| `<Profile>`    | Minimum     | Repository overview shown on the maintainer profile. |
| `<Forum>`      | Recommended | Forum thread or support landing page.                |
| `<WebPage>`    | Recommended | Primary repository or maintainer home page.          |
| `<Icon>`       | Recommended | Icon URL for the app or maintainer profile.          |
| `<Discord>`    | Advanced    | Discord invite or support community URL.             |
| `<Facebook>`   | Advanced    | Repository or maintainer Facebook page.              |
| `<Photo>`      | Advanced    | Optional repository profile image.                   |
| `<Reddit>`     | Advanced    | Repository or maintainer Reddit community URL.       |
| `<Twitter>`    | Advanced    | Repository or maintainer Twitter/X profile.          |
| `<Video>`      | Advanced    | Optional repository profile video.                   |
| `<DonateLink>` | Advanced    | Donation or sponsorship URL.                         |
| `<DonateText>` | Advanced    | Call-to-action text paired with `<DonateLink>`.      |

The submission portal may reject unsafe values or unsupported combinations even when a tag is listed here. Always rerun validation and scan at [ca.unraid.net/submit](https://ca.unraid.net/submit) before submitting.
