# Version 7.2.1 2025-11-19

Il s'agit d'une mise à jour mineure avec d'excellentes corrections et améliorations.

## Mise à niveau

Pour des instructions étape par étape, voir [Mise à jour de Unraid](../../unraid-os/system-administration/maintain-and-update/upgrading-unraid.mdx).

### Problèmes connus

Pour d'autres problèmes connus, voir les [notes de version 7.2.0](7.2.0.md#known-issues).

### Rétrogradation

Si vous revenez en arrière plus tôt que 7.2.0, consultez aussi les [notes de version 7.2.0](7.2.0.md#rolling-back).

## Changements par rapport à [7.2.0](7.2.0.md)

### webGUI

- Improvement: On _**Settings → Schedule → Mover Settings**_ add a _Disable_ option to _Mover schedule_
- Correction : Tableau de bord : Amélioration de la détection des processeurs Intel plus récents
- Correction : Le menu déroulant des agents de notification ne fonctionne pas en mode à onglets
- Correction : Erreur visuelle affichant les températures dans les notifications
- Fix: Improve search, language, and array usage on sidebar themes
- Fix: GUI Search results didn't update when plugins were uninstalled
- Fix: When editing a Share, on the _NFS Security Settings_ tab, the _Write settings to_ feature did not work for security modes Public and Secure
- Fix: _Show banner background color fade_ works again
- Fix: Reduce header background flash going from black to white on page load

### Stockage

- Amélioration : Lorsqu'un type de système de fichiers d'un pool est défini sur 'Auto', tous les emplacements de pool doivent être remplis
- Amélioration : La nouvelle configuration avec la préservation des affectations réduira les emplacements du pool pour correspondre au nombre de disques présents
- Correction : Remonter le dataset si 'zfs destroy' échoue car le dataset est occupé
- Correction : Si un partage est créé sans fichier de configuration, assurez-vous que les données ne sont stockées que sur les pools participant aux partages utilisateurs
- Correction : Info périphériques : Afficher uniquement la section 'Statut des périphériques du pool' lorsque le pool est monté
- Correction : Fonctionnalité de disque vide : les données partagées sur le(s) disque(s) cible(s) restent visibles pendant l'opération, pour plus de détails voir [Conversion vers un nouveau type de système de fichiers](../using-unraid-to/manage-storage/file-systems.mdx#converting-to-a-new-file-system-type).
  - Option '-e' supprimée du script 'mover' ; utilisez désormais le WebGUI
- Correction : Réduction de la journalisation excessive liée à l'arrêt des disques et aux auto-tests SMART
- Fix: Increase pool device limit from 120 to 200
- Fix: Suppress confusing "Share cache full" messages in syslog
- Fix: Correct /mnt/user permissions issue

### Divers

- Correction : Amélioration de la découverte WSD sur votre réseau local (pour qu'Unraid apparaisse dans le réseau Windows, anciennement voisinage réseau)
  - Générer un /boot/config/machine-id unique et l'installer sur /etc/machine-id au démarrage
  - Patch wsdd2 pour fonctionner lorsque "L'accès de l'hôte aux réseaux personnalisés" est activé
  - Remarque : la découverte ne fonctionne généralement pas si les clients utilisent Tailscale
- Correction : Mettre à jour 'lshw' pour prendre en charge les LED sur les systèmes Lincstation
- Correction : Réajouter l'agent de notification 'Join' de Unraid 6.12
- Fix: Load br\_netfilter in rc.inet1 'start' function and set default IPv4/IPv6 forwarding policy to ACCEPT (should fix VM IPv6 issues)
- Fix: Starting Docker causes all IPv6 packets on br0 to be dropped
- Fix: When stopping Samba, terminate master smbd process first
- Fix: Add support for "Intel Corporation Alder Lake-N PCH CNVi WiFi" interface

### Unraid API

- Fix: Ensure Connect features are fully disabled unless Connect is installed
- dynamix.unraid.net 4.27.0 - [voir les changements](https://github.com/unraid/api/releases)
