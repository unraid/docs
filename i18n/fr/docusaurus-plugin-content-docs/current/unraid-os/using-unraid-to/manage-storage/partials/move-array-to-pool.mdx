:::info[Common cas d'utilisation]

Déplacer les fichiers vers le %%cache pool|cache-pool%% après maintenance ou après avoir ajouté un nouvel appareil cache pour améliorer les performances.

:::

Pour déplacer des fichiers du %%array|array%% vers un pool :

1. Allez dans **Paramètres** et désactivez Docker et le %%VM|vm%% Manager pour éviter que des fichiers ouverts ne puissent interférer.
2. Dans l'onglet **Partages**, pour chaque partage que vous souhaitez déplacer (comme `appdata` ou `system`), définissez le **Stockage principal** sur le %%cache pool|cache-pool%% de destination et le **Stockage secondaire** sur le %%array|array%%. Définissez **Action du déplacer** sur **array → cache**.
3. Allez dans l'onglet **Principal** et cliquez sur **Déplacer maintenant** pour commencer à transférer les fichiers du %%array|array%% vers le %%cache pool|cache-pool%%.
4. After the %%Mover|mover%% finishes, check that the files are now on the %%cache pool|cache-pool%%.
5. Une fois le déplacement terminé, réactivez Docker et le %%VM|vm%% Manager dans **Paramètres**.
6. (Optionnel) Pour définir un partage à utiliser exclusivement avec le %%cache pool|cache-pool%% : si vous souhaitez que tous les fichiers d'un partage restent sur le %%cache pool|cache-pool%%, définissez le **Stockage principal** sur votre %%cache pool|cache-pool%% et le **Stockage secondaire** sur **Aucun** pour ce partage.
