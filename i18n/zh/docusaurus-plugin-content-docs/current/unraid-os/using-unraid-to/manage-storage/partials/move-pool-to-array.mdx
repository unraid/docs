:::info[Common 用例]

在执行维护或升级之前，从 %%cache pool|cache-pool%% 移动文件到 %%array|array%% 确保您的数据安全。

:::

将文件从池移动到 %%array|array%%：

1. 进入**设置**并关闭 Docker 和 %%VM|vm%% 管理器。这样可以防止任何文件被打开，允许 %%Mover|mover%% 顺利转移所有内容。
2. 在**共享**标签中，对于要移动的每个共享（如 `appdata` 或 `system`），将**主存储**设置为源 %%cache pool|cache-pool%%，将**次级存储**设置为 %%array|array%%。将**Mover 操作**设置为**缓存 → 阵列**。
3. 进入**主**标签并点击**立即移动**以将文件从 %%cache pool|cache-pool%% 转移到 %%array|array%%。
4. 在 %%Mover|mover%% 完成后，点击主标签中缓存条目旁边的文件夹图标检查文件是否已移动。
5. 一旦所有文件都位于 %%array|array%% 上，您就可以安全地重新打开 Docker 和 %%VM|vm%% 服务。
