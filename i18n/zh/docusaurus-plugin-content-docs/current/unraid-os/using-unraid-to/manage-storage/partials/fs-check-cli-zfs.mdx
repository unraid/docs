通过命令行检查 %%ZFS|zfs%% 文件系统：

<h4>推荐方法</h4>

%%ZFS|zfs%% 使用 **scrubbing** 替代传统文件系统检查（如 `fsck`）。 %%scrub|scrub%% 会检查池中的所有数据以验证校验和，并在冗余配置中自动修复任何发现的损坏。

- 在 **正常模式** 下启动 %%array|array%% 并运行：`zpool scrub poolname`
  - 将 `poolname` 替换为您的实际 %%ZFS|zfs%% 池名。

<h4>检查池状态</h4>
要查看当前健康状况和进行中的 %%scrub|scrub%% 进度，请使用命令：`zpool status -v poolname`

此命令显示有关池健康、任何错误和 %%scrub|scrub%% 进度的详细信息。

<h4>其他 %%ZFS|zfs%% 命令</h4>

- 暂停 %%scrub|scrub%%：`zpool scrub -p poolname`
- 停止 %%scrub|scrub%%：`zpool scrub -s poolname`
- 列出所有池：`zpool list`

:::info

%%ZFS|zfs%% 不需要也不支持传统的 `fsck` 实用程序。 %%ZFS|zfs%% 的事务性质意味着它会从一个一致的状态原子性地移动到另一个状态，使得清除成为首选的完整性检查方法。

:::
