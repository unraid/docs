# 版本 7.2.1-rc.1 2025-11-07

这是一个小更新，包含一些重要的修复和改进。

## 升级

有关分步说明，请参阅[升级 Unraid](../../unraid-os/system-administration/maintain-and-update/upgrading-unraid.mdx)。

### 已知问题

有关其他已知问题，请参见[7.2.0版本说明](7.2.0.md#known-issues)。

### 回滚

如果回滚至 7.2.0 之前的版本，请查看 [7.2.0 版本说明](7.2.0.md#rolling-back)。

## 与[7.2.0](7.2.0.md)的变更对比

### webGUI

- 修复：仪表盘：改进对较新Intel CPU的检测
- 修复：通知代理下拉菜单在标签模式中无法使用
- 修复：温度在通知中显示错误

### 存储

- 改进：当磁盘组的文件系统类型设置为“自动”时，必须填满所有磁盘槽
- 改进：新配置“保留分配”将减少磁盘槽以匹配现有磁盘数量
- 修复：如果“ZFS 销毁”因数据集繁忙失败，重新挂载数据集
- 修复：如果没有配置文件而创建共享，请确保数据仅存储在参与用户共享的磁盘组中
- 修复：设备信息：仅在磁盘组装载时显示“磁盘组设备状态”部分
- 修复：空盘功能：操作期间目标磁盘上的共享数据现保持可见，使用详情请参阅[转换到新文件系统类型](../using-unraid-to/manage-storage/file-systems.mdx#converting-to-a-new-file-system-type)。
  - Removed '-e' option from 'mover' script; use the webGUI now
- 修复：减少与硬盘休眠和SMART自检有关的过多日志记录

### 其他

- Fix: Improve WSD discovery on your local network (so Unraid appears in Windows Network, formerly Network Neighborhood)
  - /boot/config/machine-id和/etc/machine-id在启动时生成唯一值并安装
  - 修补wsdd2以在启用“主机访问自定义网络”时工作
  - 注意：如果客户端运行Tailscale，通常无法进行发现
- 修复：更新“lshw”以支持Lincstation系统上的LED灯
- Fix: Re-add the 'Join' Notification Agent from Unraid 6.12
- 修复：在rc.inet1“启动”函数中加载br\_netfiltering并将默认IPv4/IPv6转发策略设置为ACCEPT（应解决VM IPv6问题）

- wsdd2-1.8.7-x86\_64-2\_SBo\_LT.tgz（已修补，以不排除shim接口）
