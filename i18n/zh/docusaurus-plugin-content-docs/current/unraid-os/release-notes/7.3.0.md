# 版本 7.3.0 2026-05-12

此稳定版发布增加了内部启动支持、上手向导、基于 TPM 的许可、Docker 网络改进、扩展的硬件支持、ZFS 和存储修复、虚拟化更新、WebGUI 安全加固以及软件包更新。

## 升级

如需分步说明，请参阅[更新 Unraid](/unraid-os/updating-unraid/)。有关您的[许可证](/unraid-os/troubleshooting/licensing-faq/)的问题？

### 已知问题 {/* #known-issues */}

- 在内部启动设置期间，WebGUI 在内部启动配置时可能会显示 **Array Offline**。在此过程中，**请勿**手动重启或移除您的闪存盘。
- 内部启动需要可在启动期间由内置 Linux 驱动访问的启动存储。需要第三方驱动或启动后配置的设备不能用于内部启动。Unraid 支持以下用于内部启动的驱动：
  - ahci
  - mmc
  - mpt3sas
  - mvsas
  - nvme
  - smartpqi
  - USB（`usb-storage` 和 `uas`，也称为 UASP）
- 如果双设备启动池包含一个带有未格式化数据分区的设备，移除其中一个启动池设备可能无法正确完成。
- 如果移除一个拆分启动池设备后立即添加另一设备，槽位数量可能不会立即更新，直到服务器重启后才会更新。
- 某些 UPS 设备可能会报告 **Online No battery detected**，并重复记录 `Event data for report 22 was too short` 消息。如果受影响，NUT 插件可能会提供可用的 UPS 监控路径。
- 某些位于 VLAN 网络上的 Docker 容器在从 7.2.4 或 7.2.5 升级后，可能无法获取 DHCP，或无法使用预期的静态网络配置。此版本为该问题提供了可能的缓解措施。

### 笔记

- 在 ZFS 启动池中添加、移除和替换设备可正常工作。如果您先使用两个小型设备创建一个专用启动池，然后逐个用更大的设备替换它们，池可能会转换为分裂池。
- 当添加、移除或替换启动池设备时，Unraid 当前不会自动更新服务器 BIOS 启动优先级。完成这些更改后，请手动验证 BIOS 启动顺序。

### 回滚 {/* #rolling-back */}

- 启用内部启动或基于 TPM 的授权后，请勿回滚到 `7.2.6` 或更早版本。
- 如果您升级 ZFS 池功能，较早的 Unraid 版本可能无法导入这些池。
- 如果需要回滚，请在版本之间切换之前，先按照正常的回滚流程操作。

如果您正在考虑回滚到 `7.3.0` 之前的任何版本，也请参阅 [7.2.6 发行说明](/unraid-os/release-notes/7.2.6/)。

## 相较于 [7.2.6](/unraid-os/release-notes/7.2.6/) 的变更

### 安全

- 安全：为已认证会话的请求处理添加 WebGUI 安全加固。此修复已包含在 7.2.5 中。
- 安全：更新软件包
- 安全说明：随附的 7.3 内核修补了两个本地权限提升漏洞：Copy Fail（CVE-2026-31431）和 Dirty Frag（CVE-2026-43284 和 CVE-2026-43500）

### 引导向导和内部启动

新用户会自动进入引导向导，在其中选择语言、时区、主题、激活码设置和启动选项，包括内部启动。

现有用户可前往 ***工具 → 入门向导*** 查看设置并迁移到内部启动。

For details, see [Onboarding](/unraid-os/getting-started/set-up-unraid/deploy-and-configure-unraid-os/).

### 许可

所有新的和替换的密钥都会尽可能使用基于 TPM 的许可，因为它比基于闪存的许可更可靠。

To proactively switch to TPM-based licensing, see [How can I move from flash-based licensing to TPM-based licensing](/unraid-os/troubleshooting/tpm-licensing-faq/#tpm).

### 容器 / Docker

- 改进：将 Docker 从 `29.3.1` 更新到 `29.4.3`。
- 新增：为需要在重启后保持稳定网络标识的容器，在 Docker 模板中添加可选的固定 MAC 地址字段，包括 DHCP 保留、路由器或防火墙规则、交换机 ACL 以及监控。
- 改进：在高级视图中将 Docker 容器的 MAC 地址与现有网络和 IP 信息一并显示。
- 修复：在安全的情况下，将 Extra Parameters 中旧的 `--mac-address=` 值迁移到新的固定 MAC 字段；如果网络仍由 Extra Parameters 管理，则保持模板不变。
- 修复：为受影响系统上的 Strix Point iGPU Docker 工作负载补充所需的缺失支持。请参阅[用户报告](https://product.unraid.net/p/strix-point-igpu-support)。
- 修复：在“添加容器”页面添加 Docker 容器时保持 WebGUI 响应。

### 存储

- 新增：在 ZFS 池状态中显示损坏文件。
- 新增：在 ***设置 → 磁盘设置 → 可调项 (zfs_arc_max)*** 中添加面向用户的 ZFS ARC 最大大小控制项。此前这是通过自定义 ZFS 驱动程序参数控制的。你可能需要将设置迁移到新位置。
- 改进：提升池和硬盘健康信息的可见性。
- 改进：增强对重复硬盘的检测及相关提示信息。
- 修复：在奇偶校验交换期间的奇偶校验复制过程中，防止硬盘停转。
- 修复：修正受影响情况下 ZFS 池每 24 小时唤醒一次的问题。
- 修复：更正当丢弃的设备被标记为 `REMOVED` 时的 ZFS 池替换处理，因此替换流程不再会使池处于降级状态，并残留过时的蓝色新设备状态或重复的覆盖警告。
- 修复：防止创建名称使用保留 ZFS 名称前缀（如 `mirror`、`raidz`、`draid` 或 `spare`）的 ZFS 池。

### WebGUI / 系统

- 改进：为 7.3 的内部启动和硬件支持工作刷新 WebGUI 和系统设备工作流。
- 改进：添加对 Cooler Master Qube 500 机箱型号的支持。
- 新增：当可用有效的机箱序列号时，在系统信息中显示机箱序列号。
- 改进：当未检测到有效的 TPM 或闪存授权设备时，显示更清晰的授权状态消息。
- 修复：在 dmidecode 单位标签变更后，修正 RAM 显示解析。
- 修复：修正受影响地区的时区标签和标识符问题。
- 修复：阻止受影响系统上的 WebGUI 和 API 请求唤醒整个 HDD 阵列。
- 修复：将内置 `rclone` 配置保存在持久位置，以便在重启后仍能保留。
- 修复：在使用会阻止 80 端口上 CONNECT 的 HTTP 代理时，修正严格的代理连通性检查。
- 修复：修正 rc.crond 状态中的拼写错误。
- 修复：修正 Discord 通知格式问题。
- 修复：更正 FTP 帮助文本，使其明确说明 FTP 默认已禁用。
- 修复：在各种文本中使用 online / offline 替代 on-line / off-line。
- 修复：移除过时的 ReiserFS 警告和残留引用。

### 文件管理器

- 改进：优化文件管理器界面和整体性能。
- 改进：优化同文件系统内的移动操作。
- 改进：优化上传行为。
- 修复：修正主标签页上的文件管理器样式，使其遵循所选主题。
- 修复：修正文件路径中涉及 `/sub/` 的路径处理问题。
- 修复：修正复制或移动操作期间的滚动条交互问题。
- 修复：修正目录名称中双引号的处理。
- 修复：修正移动和重命名的边缘情况。
- 修复：在受影响的重命名操作期间保留空目录。

### 网络 / 硬件

- 改进：更新内核配置以支持额外的 AMD XDNA 和 ACP，包括 `CONFIG_DRM_ACCEL_AMDXDNA`、`CONFIG_DRM_AMD_ACP`、`CONFIG_REMOTEPROC`、通用 power-domain 支持以及 `CONFIG_REMOTEPROC_CDEV`。
- 改进：添加 AMD NPU 固件、Intel 蓝牙固件、Intel 无线固件以及 `amdxdna` 内核模块。
- 新增：添加 ***设置 → Tailscale*** 占位页面，以便更容易发现插件。

### 虚拟化

- 改进：提升系统设备可见性和虚拟机模板工作流。
- 改进：优化自定义 VNC 端口验证和默认值。
- 改进：将 QEMU 更新到 `10.2.2`、libvirt 更新到 `12.2.0`，并刷新 OVMF 固件包。
- 修复：修正虚拟机快照提交清理未能正确更新快照元数据的问题。
- 修复：防止由用户自定义模板创建的虚拟机继承源虚拟机的 MAC 地址。
- 修复：在启动期间更新已弃用的 VM 机器类型，使较旧的 VM 在升级后不会因不受支持的 QEMU 机器类型错误而失败。
- 修复：在使用 Virtio 3D 时，在 VM 列表中显示第一个 VM 的已分配 `RenderGPU`。

### Linux kernel

- Linux 内核：更新至 `6.18.29-Unraid`。
