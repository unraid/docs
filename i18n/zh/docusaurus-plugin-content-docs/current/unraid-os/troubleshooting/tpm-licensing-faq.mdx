# TPM 许可常见问题解答（7.3+）

本页包含有关 Unraid OS 中基于 TPM 的许可的常见问题。有关一般许可问题，请参阅 [许可常见问题解答](/unraid-os/troubleshooting/licensing-faq/)。有关内部启动问题，请参阅 [内部启动常见问题解答（7.3+）](/unraid-os/getting-started/set-up-unraid/internal-boot-faq/)。

---

### 如何从基于闪存的许可迁移到基于 TPM 的许可？ {/* #tpm */}

:::note[Unraid 7.3 and later]

Unraid 7.3 及更高版本提供基于 TPM 的许可。

:::

所有新的和替换的密钥都会尽可能使用基于 TPM 的许可，因为它比基于闪存的许可更可靠。

要主动切换到基于 TPM 的许可，请转到 ***工具 → 注册***，然后点击 **将许可证移至 TPM**。系统会跳转到 Unraid 账户网站，您可以在其中批准替换请求。

在点击 **将许可证移至 TPM** 后，Unraid 账户流程会显示 **替换许可证密钥** 屏幕：

![点击将许可证移至 TPM 后显示的替换许可证密钥屏幕](/img/move-license-to-tpm-replace-key.png)

如果 **将许可证移至 TPM** 按钮不可用，则表示 Unraid 无法检测到您的 TPM 2.0 设备。它可能在 BIOS 中被禁用了。检查与 **Trusted Computing** 相关的设置，并保持 **Secure Boot** 处于禁用状态。

### 迁移到基于 TPM 的许可有哪些好处？ {/* #tpm-benefits */}

传统 USB 启动设备对某些用户来说正变得不太可靠。基于 TPM 的许可意味着即使您更改启动硬件，许可证仍然有效。

### 如果我换了主板会怎样？ {/* #tpm-new-motherboard */}

您可以在新主板上替换基于 TPM 的许可证，类似于在新的 USB 设备上替换基于闪存的许可证。在这种情况下，仅原始许可证会被列入黑名单，而不是整块主板。

### 如果我更新 BIOS 会怎样？我需要新的许可证吗？ {/* #tpm-bios */}

TPM 不应受到 BIOS 更新的影响。如果不知何故受到了影响，您可以像获得[新主板](#tpm-new-motherboard)一样替换您的许可证。

### 如果我的主板没有 TPM 设备怎么办？ {/* #tpm-missing */}

切换到基于 TPM 的许可并不是硬性要求，但许多较旧的主板都带有一个可安装 TPM 附加设备的接口。这些设备价格低廉且广泛可用。

### 在从闪存启动时，我可以切换到基于 TPM 的许可吗？ {/* #tpm-flash-boot */}

可以，许可方式与启动方式无关。

有关内部启动相关问题，请参阅 [内部启动常见问题解答（7.3+）](/unraid-os/getting-started/set-up-unraid/internal-boot-faq/)。

### 我可以在 Unraid 虚拟机中使用 TPM 吗？ {/* #tpm-vm */}

不支持。

### 如果我需要从硬件故障中恢复，我可以在 TPM 和 USB 之间移动我的许可证吗？ {/* #tpm-usb-replacement */}

是。

迁移到基于 TPM 的许可会将原始 USB 设备上的许可证列入黑名单。之后：

- 您可以使用许可证替换流程切换回 USB 设备。
- 您可以使用 TPM 替换流程从一块带 TPM 的主板切换到另一块。

如果您遇到导致常规替换流程无法进行的硬件故障，请联系 [Unraid 支持](https://support.unraid.net/support/home) 寻求帮助。

### TPM 重置会影响我的 Unraid 许可证吗？ {/* #tpm-reset */}

不会。Unraid TPM 许可证使用您的 TPM 标识符，因此 TPM 重置不应影响您的 Unraid 许可证。
