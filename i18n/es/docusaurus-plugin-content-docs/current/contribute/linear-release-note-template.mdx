## Reglas de redacción

- Lidere con el impacto para el usuario. Convierta las notas de ingeniería en lo que cambió para los usuarios, los administradores o la seguridad de actualización.
- Mantenga las viñetas concisas y objetivas. No incluya nombres internos de proyectos, IDs de incidencias, nombres de ramas, hashes de commits ni detalles de implementación, a menos que sean importantes para los usuarios.
- Use los prefijos `New:`, `Improvement:`, `Fix:` y `Security:` de forma coherente en las secciones de cambios.
- Agrupe los elementos relacionados bajo el encabezado más específico. Use `WebGUI / System` para cambios amplios de la interfaz de usuario, ajustes, servicio del sistema, estado de licencias y notificaciones.
- Incluya una sección `BREAKING CHANGES` solo cuando los usuarios deban cambiar su comportamiento o cuando cambie el comportamiento de reversión, compatibilidad, red, almacenamiento, datos o configuración.
- No invente CVE, paquetes afectados, versiones de paquetes, advertencias de reversión, problemas conocidos ni recomendaciones de actualización. Si el material de origen no está claro, escriba `[NEEDS CONFIRMATION: ...]`.
- Conserve los enlaces públicos de origen cuando ayuden a los usuarios a verificar detalles, como notas de la versión aguas arriba, informes de usuarios o documentación relacionada.
- Prefiera los enlaces de documentación en este formato: `/unraid-os/release-notes/[VERSION]/`.
- Use **negrita** para las etiquetas de la interfaz y ***negrita en cursiva*** para las rutas de navegación, como ***Settings → Disk Settings***.
- Use código en línea para nombres de paquetes, comandos, claves de configuración, rutas, versiones, símbolos de configuración del kernel y cadenas de error.
- Elimine las secciones opcionales no usadas antes de publicar.

## Lista de verificación de notas de la versión

Antes de publicar, confirme que:

- El encabezado es `# Version [VERSION] [YYYY-MM-DD]`.
- El resumen menciona los cambios más importantes sin repetir todas las secciones.
- Las notas de actualización, de problemas conocidos y de reversión son precisas para esta versión.
- Las afirmaciones de seguridad coinciden con los avisos confirmados o los registros de cambios de los paquetes.
- Los conteos de paquetes coinciden con las listas de paquetes.
- Los enlaces de la versión anterior apuntan a la versión correcta.
- Las secciones opcionales con marcadores de posición se han eliminado.
