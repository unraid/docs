# Actualización de Unraid

Actualizar el sistema operativo Unraid garantiza que dispongas de las características más recientes, actualizaciones de seguridad y soporte para hardware. Esta página describe el proceso estándar de actualización, junto con consejos para solucionar problemas y opciones de actualización manual.

:::note\[Prerequisites]

Antes de comenzar la actualización, asegúrate de crear una copia de seguridad completa de tu dispositivo USB. Para más detalles, consulta [Haciendo una copia de seguridad de tu dispositivo](../maintain-and-update/changing-the-flash-device.mdx#backing-up-your-flash-device).

:::

:::tip[Best prácticas antes de actualizar]

1. **Hacer una copia de seguridad** de su unidad flash y cualquier dato importante.
2. **Lea las notas de la versión** para la versión que planea instalar.
3. **Actualice todos los complementos** a sus versiones más recientes.
4. **Opcionalmente, detenga el array** antes de proceder.
5. **Actualice Unraid OS** utilizando la página ***Herramientas → Actualizar OS***.
6. **Reinicie** su servidor para completar la actualización.

:::

:::caution[ZFS advertencias de actualización del grupo]

Al actualizar a Unraid 7.x, puedes ver advertencias sobre las actualizaciones de características del grupo [%%ZFS|zfs%%](../../advanced-configurations/optimize-storage/zfs-storage.mdx) durante el inicio o en el %%WebGUI|web-gui%%. Estas advertencias no son una señal de problema; simplemente indican que tu grupo %%ZFS|zfs%% está utilizando características de una versión anterior de ZFS.

No es urgente actualizar tu grupo, pero si decides hacerlo, ten en cuenta que podría no funcionar con versiones anteriores de Unraid. Esto significa que es posible que no puedas volver a una versión anterior de Unraid después de actualizar.

Como siempre, recuerda hacer una copia de seguridad de tus datos antes de actualizar tus grupos de %%ZFS|zfs%%.

:::

<details>
  <summary><strong>Las VMs muestran "no se puede obtener la interfaz MTU" o errores de red</strong> - Haga clic para expandir/colapsar</summary>

  1. Escriba `fs0:`.
  2. Luego escriba `cd efi/boot`.
  3. Finalmente, escriba `bootx64.efi`.
  4. Navegue a la pestaña **Aplicaciones**, **Aplicaciones anteriores** y marque lo que desea reinstalar y haga clic en **"Instalar xx Aplicaciones"**.
</details>

  2. Configure el puente de red en `br0` y haga clic en **Aplicar**.
  3. Navegue a ***Configuración → Administrador de VM*** (en **Vista avanzada**) y configure el puente predeterminado en `br0`.

## Degradación manual

Las degradaciones manuales son raramente necesarias pero pueden serlo si no puedes acceder al %%WebGUI|web-gui%% o necesitas volver a una versión anterior. Antes de proceder, es importante que hagas una copia de seguridad de tu dispositivo USB, cuyos detalles puedes encontrar en [Copia de seguridad de tu dispositivo flash](../maintain-and-update/changing-the-flash-device.mdx#backing-up-your-flash-device).

:::info[What sucede durante el proceso de degradación]

Al usar la herramienta Degradar OS (WebGUI), la herramienta manejará automáticamente lo siguiente:

1. **Validación del sistema**: Comprueba que la degradación sea segura de realizar.
2. **Copia de seguridad de archivos**: Se creará una copia de seguridad de tus archivos del sistema actuales.
3. **Verificación de la versión**: La herramienta se asegura de que los archivos para la versión objetivo sean válidos y completos.
4. **Reemplazo de archivos**: Reemplaza los archivos actuales del sistema operativo con los archivos de la versión anterior.
5. **Preservación de configuraciones**: Se mantendrán tus configuraciones actuales.
6. **Preparación del sistema**: Se preparará el sistema para la versión anterior.
7. **Programación de reinicio**: Se programará un reinicio para finalizar la degradación.

:::

#### Verificación posterior a la degradación

Después de que la degradación esté completa y tu servidor haya reiniciado:

1. **Comprueba la versión**: Verifica que estés usando la versión de Unraid que esperas.
2. **Prueba la funcionalidad principal**: Asegúrate de que los servicios de %%array|array%%, Docker y VM estén funcionando correctamente.
3. **Verifica los complementos**: Comprueba que tus complementos instalados sean compatibles y funcionen como se espera.
4. **Prueba los contenedores de Docker**: Inicia tus contenedores de Docker y confirma que estén funcionando correctamente.
5. **Revisa las VMs**: Si tienes máquinas virtuales (VMs), asegúrate de que inicien y operen correctamente. Consulta [Configuración de VM](../../using-unraid-to/create-virtual-machines/vm-setup.mdx) para orientación sobre configuración.
6. **Monitorea los registros del sistema**: Mantente atento a los registros del sistema para detectar mensajes de error. Utiliza [herramientas de diagnóstico](../../troubleshooting/diagnostics/capture-diagnostics-and-logs.mdx) para capturar información detallada del sistema.

#### Resolución de problemas de degradación

<details>
  <summary><strong>Opción de degradación no disponible</strong> - Haz clic para expandir/colapsar</summary>

  Si no ves una opción para degradar:

  - **Revisa el historial del sistema**: Asegúrate de haber mejorado previamente desde una versión compatible.
  - **Verifica la integridad de los archivos**: Los archivos de la versión anterior pueden estar dañados o desaparecer.
  - **Verifica la compatibilidad**: La versión anterior puede no ser compatible con tu hardware actual.
  - **Método manual**: Si es necesario, sigue los métodos de degradación manual que se describen en esta página.
</details>

<details>
  <summary><strong>La degradación falla durante el proceso</strong> - Haz clic para expandir/colapsar</summary>

  Si el proceso de degradación no sale como se planeó:

  1. **No te alarmes**: Tu sistema debería seguir funcionando con la versión actual.
  2. **Revisa los registros**: Examina los registros del sistema para encontrar mensajes de error específicos. Utiliza [herramientas de diagnóstico](../../troubleshooting/diagnostics/capture-diagnostics-and-logs.mdx) para un análisis de registros completo.
  3. **Verifica los archivos**: Asegúrate de que todos los archivos necesarios para la versión objetivo estén disponibles.
  4. **Intenta el método manual**: Si la herramienta no funciona, considera usar los métodos de degradación manual.
  5. **Contacta al soporte**: Si los problemas persisten, comunícate con el soporte de Unraid con tus diagnósticos.
</details>

<details>
  <summary><strong>El sistema no arranca después de la degradación</strong> - Haz clic para expandir/colapsar</summary>

  Si tu sistema no arranca después de la degradación:

  1. **Arranca desde USB**: Usa otra computadora para acceder a tu dispositivo flash USB. Consulta [resolución de problemas de arranque](../../troubleshooting/common-issues/boot-and-startup-failures.mdx) para obtener pasos detallados de recuperación.
  2. **Restaura desde una copia de seguridad**: Copia tus archivos de respaldo de nuevo al dispositivo USB.
  3. **Revisa la integridad de los archivos**: Asegúrate de que todos los archivos del sistema estén presentes y no estén dañados.
  4. **Intenta el Modo Seguro**: Arranca en [Modo Seguro](../../using-unraid-to/customize-your-experience/plugins.mdx#troubleshooting-with-safe-mode) para más resolución de problemas.
  5. **Recuperación manual**: Si es necesario, utiliza los métodos de recuperación manual.
</details>

<details>
  <summary><strong>Complementos o contenedores de Docker no funcionarán</strong> - Haz clic para expandir/colapsar</summary>

  Después de degradar, algunos complementos o contenedores pueden no funcionar correctamente:

  - **Actualiza los complementos**: Busca actualizaciones de complementos compatibles con la versión degradada. Revisa [Aplicaciones Comunitarias](../../using-unraid-to/run-docker-containers/community-applications.mdx) para encontrar versiones compatibles.
  - **Reconstruye Docker**: Elimina y recrea tu archivo de imagen de Docker. Consulta [resolución de problemas de Docker](../../troubleshooting/common-issues/docker-troubleshooting.mdx#re-creating-the-docker-image-file) para pasos detallados.
    - **Revisa las dependencias**: Asegúrate de que todas las dependencias necesarias estén disponibles.
    - **Compatibilidad de complementos**: Algunos complementos podrían no ser compatibles con versiones anteriores de Unraid.
    - **Plugins dependientes de características de UI/WebGUI**: Los complementos que dependen de las nuevas características de WebGUI pueden no funcionar correctamente en versiones anteriores.
    - **Plugins temáticos**: Es posible que los complementos de personalización visual necesiten actualizaciones o eliminación si son incompatibles.
</details>
