# Versión 7.2.1 2025-11-19

Esta es una pequeña actualización con algunas excelentes correcciones y mejoras.

## Actualización

Para instrucciones paso a paso, consulte [Actualización de Unraid](../../unraid-os/system-administration/maintain-and-update/upgrading-unraid.mdx).

### Problemas conocidos

Para otros problemas conocidos, vea las [notas de la versión 7.2.0](7.2.0.md#known-issues).

### Deshacer cambios

Si revierte a una versión anterior a 7.2.0, también consulte las [notas de la versión 7.2.0](7.2.0.md#rolling-back).

## Cambios versus [7.2.0](7.2.0.md)

### webGUI

- Improvement: On _**Settings → Schedule → Mover Settings**_ add a _Disable_ option to _Mover schedule_
- Corrección: Panel de control: Mejor detección de CPUs Intel más nuevos
- Corrección: Dropdown de Agentes de Notificación no funciona en modo de pestañas
- Corrección: Error visual que muestra temperaturas en notificaciones
- Fix: Improve search, language, and array usage on sidebar themes
- Fix: GUI Search results didn't update when plugins were uninstalled
- Fix: When editing a Share, on the _NFS Security Settings_ tab, the _Write settings to_ feature did not work for security modes Public and Secure
- Fix: _Show banner background color fade_ works again
- Fix: Reduce header background flash going from black to white on page load

### Almacenamiento

- Mejora: Cuando el Tipo de Sistema de Archivos de un pool se establece en 'Auto', todos los espacios del pool deben estar llenos
- Mejora: Nueva Configuración con Preservar Asignaciones reducirá los espacios del pool para coincidir con el número de discos presentes
- Corrección: Volver a montar dataset si 'ZFS destroy' falla porque el dataset está ocupado
- Corrección: Si se crea un share sin un archivo de configuración, asegúrese de que los datos solo se almacenen en pools que participen en comparticiones de usuario
- Corrección: Solo mostrar la sección 'Estado del Dispositivo del Pool' cuando el pool está montado
- Corrección: Función de Disco Vacío: los datos compartidos en el disco(s) objetivo ahora permanecen visibles durante la operación, consulta [Convirtiendo a un nuevo tipo de sistema de archivos](../using-unraid-to/manage-storage/file-systems.mdx#converting-to-a-new-file-system-type) para detalles de uso.
  - Eliminado la opción '-e' del script 'mover'; ahora use el WebGUI
- Corrección: Reducir registro excedente relacionado con la desactivación de discos y auto pruebas SMART
- Fix: Increase pool device limit from 120 to 200
- Fix: Suppress confusing "Share cache full" messages in syslog
- Fix: Correct /mnt/user permissions issue

### Varios

- Corrección: Mejorar el descubrimiento WSD en su red local (para que Unraid aparezca en la Red de Windows, anteriormente Barrio de Red)
  - Generar único /boot/config/machine-id e instalar en /etc/machine-id al iniciar
  - Parchear wsdd2 para que funcione cuando "Acceso del host a redes personalizadas" está habilitado
  - Nota: el descubrimiento típicamente no funciona si los clientes están ejecutando Tailscale
- Corrección: Actualizar 'lshw' para soportar LEDs en sistemas Lincstation
- Corrección: Reagregar el Agente de Notificación 'Join' de Unraid 6.12
- Fix: Load br\_netfilter in rc.inet1 'start' function and set default IPv4/IPv6 forwarding policy to ACCEPT (should fix VM IPv6 issues)
- Fix: Starting Docker causes all IPv6 packets on br0 to be dropped
- Fix: When stopping Samba, terminate master smbd process first
- Fix: Add support for "Intel Corporation Alder Lake-N PCH CNVi WiFi" interface

### API de Unraid

- Fix: Ensure Connect features are fully disabled unless Connect is installed
- dynamix.unraid.net 4.27.0 - [ver cambios](https://github.com/unraid/api/releases)
