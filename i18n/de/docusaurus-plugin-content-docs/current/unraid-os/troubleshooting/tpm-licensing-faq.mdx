# TPM-Lizenz-FAQ (7.3+)

Diese Seite enthält häufig gestellte Fragen zur TPM-basierten Lizenzierung in Unraid OS. Allgemeine Fragen zur Lizenzierung finden Sie in der [Lizenz-FAQ](/unraid-os/troubleshooting/licensing-faq/). Fragen zum internen Booten finden Sie in der [FAQ zum internen Booten (7.3+)](/unraid-os/getting-started/set-up-unraid/internal-boot-faq/).

---

### Wie kann ich von der flashbasierten Lizenzierung zur TPM-basierten Lizenzierung wechseln? {/* #tpm */}

:::note[Unraid 7.3 und höher]

Die TPM-basierte Lizenzierung ist in Unraid 7.3 und höher verfügbar.

:::

Alle neuen und Ersatzschlüssel verwenden nach Möglichkeit TPM-basierte Lizenzierung, weil sie zuverlässiger ist als Flash-basierte Lizenzierung.

Um proaktiv zur TPM-basierten Lizenzierung zu wechseln, gehen Sie zu ***Tools → Registrierung*** und klicken Sie auf **Lizenz auf TPM verschieben**. Sie werden auf die Unraid-Account-Website weitergeleitet, wo Sie die Ersetzungsanfrage genehmigen können.

Nach dem Klicken auf **Lizenz auf TPM verschieben** zeigt der Unraid-Account-Ablauf den Bildschirm **Lizenzschlüssel ersetzen**:

![Bildschirm „Lizenzschlüssel ersetzen“ nach dem Klicken auf „Lizenz auf TPM verschieben“](/img/move-license-to-tpm-replace-key.png)

Wenn die Schaltfläche **Lizenz auf TPM verschieben** nicht verfügbar ist, konnte Unraid Ihr TPM-2.0-Gerät nicht erkennen. Es ist möglicherweise im BIOS deaktiviert. Prüfen Sie die Einstellungen für **Trusted Computing** und lassen Sie **Secure Boot** deaktiviert.

### Was sind die Vorteile des Wechsels zur TPM-basierten Lizenzierung? {/* #tpm-benefits */}

Herkömmliche USB-Bootgeräte werden für einige Benutzer zunehmend unzuverlässig. TPM-basierte Lizenzierung bedeutet, dass Ihre Lizenz gültig bleibt, auch wenn Sie die Boot-Hardware ändern.

### Was ist, wenn ich ein neues Mainboard bekomme? {/* #tpm-new-motherboard */}

Sie können eine TPM-basierte Lizenz auf einem neuen Mainboard ersetzen, ähnlich wie beim Ersetzen einer flashbasierten Lizenz auf einem neuen USB-Gerät. In diesem Fall wird nur die ursprüngliche Lizenz auf die schwarze Liste gesetzt, nicht das gesamte Mainboard.

### Was passiert, wenn ich mein BIOS aktualisiere? Benötige ich eine neue Lizenz? {/* #tpm-bios */}

TPM sollte von BIOS-Updates nicht betroffen sein. Falls doch, können Sie Ihre Lizenz ersetzen, als hätten Sie ein [neues Mainboard](#tpm-new-motherboard).

### Was ist, wenn mein Mainboard kein TPM-Gerät hat? {/* #tpm-missing */}

Der Wechsel zur TPM-basierten Lizenzierung ist keine harte Voraussetzung, aber viele ältere Mainboards verfügen über einen Header, an dem ein TPM-Erweiterungsgerät installiert werden kann. Diese Geräte sind kostengünstig und weit verbreitet.

### Kann ich zur TPM-basierten Lizenzierung wechseln, während ich von Flash boote? {/* #tpm-flash-boot */}

Ja, die Lizenzierungsmethode ist nicht an die Bootmethode gebunden.

Für Fragen zum internen Booten finden Sie die [FAQ zum internen Booten (7.3+)](/unraid-os/getting-started/set-up-unraid/internal-boot-faq/).

### Kann ich TPM innerhalb einer Unraid-VM verwenden? {/* #tpm-vm */}

Nein, das wird nicht unterstützt.

### Kann ich meine Lizenz zwischen TPM und USB verschieben, wenn ich mich von einem Hardwareausfall erholen muss? {/* #tpm-usb-replacement */}

Ja.

Der Wechsel zur TPM-basierten Lizenzierung setzt die Lizenz auf dem ursprünglichen USB-Gerät auf die schwarze Liste. Danach gilt:

- Sie können mithilfe des Lizenzersetzungsablaufs zu einem USB-Gerät zurückwechseln.
- Sie können mithilfe des TPM-Ersetzungsablaufs von einem TPM-ausgestatteten Mainboard zu einem anderen wechseln.

Wenn Sie einen Hardwareausfall haben, der den normalen Ersetzungsablauf verhindert, wenden Sie sich bitte an den [Unraid-Support](https://support.unraid.net/support/home), um Hilfe zu erhalten.

### Wird ein TPM-Reset meine Unraid-Lizenz beeinflussen? {/* #tpm-reset */}

Nein. Die Unraid-TPM-Lizenz verwendet Ihre TPM-Kennungen, daher sollten TPM-Resets Ihre Unraid-Lizenz nicht beeinflussen.
