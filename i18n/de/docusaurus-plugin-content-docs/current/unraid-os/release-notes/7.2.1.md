# Version 7.2.1 19.11.2025

Dies ist ein kleines Update mit einigen großartigen Korrekturen und Verbesserungen.

## Upgrade

Für Schritt-für-Schritt-Anleitungen siehe [Unraid aktualisieren](../../unraid-os/system-administration/maintain-and-update/upgrading-unraid.mdx).

### Bekannte Probleme

Für weitere bekannte Probleme siehe die [7.2.0 Release Notes](7.2.0.md#known-issues).

### Rückschritte

Wenn Sie auf eine Version vor 7.2.0 zurückgehen, schauen Sie sich auch die [7.2.0 Release Notes](7.2.0.md#rolling-back) an.

## Änderungen vs. [7.2.0](7.2.0.md)

### webGUI

- Verbesserung: Auf _**Einstellungen → Zeitplan → Mover-Einstellungen**_ eine _Deaktivieren_ Option zum _Mover-Zeitplan_ hinzufügen
- Korrektur: Dashboard: Verbesserung der Erkennung neuerer Intel-CPUs
- Korrektur: Das Dropdown für Benachrichtigungsagenten funktioniert im Registerkartenmodus nicht
- Korrektur: Visueller Fehler beim Anzeigen von Temperaturen in Benachrichtigungen
- Fix: Suche, Sprache und Array-Nutzung in Sidebar-Themen verbessern
- Fix: GUI-Suchergebnisse wurden nicht aktualisiert, wenn Plugins deinstalliert wurden
- Fix: Beim Bearbeiten eines Shares auf dem Reiter _NFS-Sicherheitseinstellungen_ funktionierte die Funktion _Einstellungen schreiben auf_ nicht für die Sicherheitsmodi Öffentlich und Sicher
- Fix: _Banner-Hintergrundfarbe Verblassen anzeigen_ funktioniert wieder
- Fehlerbehebung: Reduzierung des Header-Hintergrund-Flash-Effekts beim Laden der Seite von Schwarz auf Weiß

### Speicher

- Verbesserung: Wenn der Dateisystemtyp eines Pools auf „Auto“ gesetzt ist, müssen alle Pool-Slots besetzt sein
- Verbesserung: Neue Konfiguration mit Beibehaltungszuweisung reduziert die Pool-Slots, um die Anzahl der vorhandenen Festplatten anzupassen
- Korrektur: Dataset erneut einhängen, wenn 'zfs destroy' fehlschlägt, weil Dataset beschäftigt ist
- Korrektur: Wenn ein Freigabe ohne Konfigurationsdatei erstellt wird, sicherstellen, dass Daten nur auf Pools gespeichert werden, die an Benutzerfreigaben teilnehmen
- Korrektur: DeviceInfo: Abschnitt „Pool-Gerätestatus“ nur anzeigen, wenn der Pool eingehängt ist
- Korrektur: Funktion „Leerdisk“: Freigabedaten auf Zieldisk(s) bleiben während des Betriebs sichtbar, siehe [Konvertierung zu einem neuen Dateisystemtyp](../using-unraid-to/manage-storage/file-systems.mdx#converting-to-a-new-file-system-type) für Verwendungsdetails.
  - Entfernt: Option '-e' aus dem 'mover' Skript; verwenden Sie jetzt das WebGUI
- Korrektur: Übermäßiges Protokollieren in Bezug auf das Herunterdrehen der Festplatten und SMART-Selbsttests reduzieren
- Fehlerbehebung: Erhöhung des Pool-Gerätelimits von 120 auf 200
- Fehlerbehebung: Unterdrückung verwirrender "Share cache full"-Meldungen im Syslog
- Fehlerbehebung: Korrektur des Berechtigungsproblems unter /mnt/user

### Verschiedenes

- Korrektur: WSD-Erkennung in Ihrem lokalen Netzwerk verbessern (damit Unraid im Windows-Netzwerk, ehemals Netzwerkumgebung, angezeigt wird)
  - Generieren einer eindeutigen /boot/config/machine-id und Installation in /etc/machine-id beim Booten
  - Patch wsdd2, um zu funktionieren, wenn "Hostzugriff auf benutzerdefinierte Netzwerke" aktiviert ist
  - Hinweis: Die Erkennung funktioniert normalerweise nicht, wenn Clients Tailscale verwenden
- Korrektur: 'lshw' aktualisieren, um LEDs auf Lincstation-Systemen zu unterstützen
- Korrektur: Den Benachrichtigungsagenten 'Join' aus Unraid 6.12 wieder hinzufügen
- Fehlerbehebung: Laden von br\_netfilter in der Funktion rc.inet1 'start' und Festlegen der Standardweiterleitungspolicy für IPv4/IPv6 auf ACCEPT (sollte IPv6-Probleme in VMs beheben)
- Fehlerbehebung: Starten von Docker führt dazu, dass alle IPv6-Pakete auf br0 verworfen werden
- Fehlerbehebung: Beim Stoppen von Samba zuerst den Hauptprozess smbd beenden
- Fehlerbehebung: Unterstützung für die "Intel Corporation Alder Lake-N PCH CNVi WiFi"-Schnittstelle hinzufügen

### Unraid API

- Fehlerbehebung: Sicherstellen, dass die Connect-Funktionen vollständig deaktiviert sind, es sei denn, Connect ist installiert
- dynamix.unraid.net 4.27.0 - [Änderungen ansehen](https://github.com/unraid/api/releases).
