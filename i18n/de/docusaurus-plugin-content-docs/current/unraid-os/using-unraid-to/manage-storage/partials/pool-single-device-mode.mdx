In **single device mode**, your %%cache pool|cache-pool%% consists of only one device. This means:

- **No redundancy:** If the device fails, any data stored on it that hasn't been transferred to your %%array|array%% will be lost.
- **Dateisystemoptionen:** Sie können andere Dateisysteme als %%BTRFS|btrfs%% verwenden, wie %%XFS|xfs%% oder %%ZFS|zfs%%.

:::important
Wenn Sie in einem Einzelgerät-Modus ein Dateisystem wählen, das nicht %%BTRFS|btrfs%% oder %%ZFS|zfs%% ist, und später weitere Geräte zum Pool hinzufügen möchten, müssen Sie das Gerät zunächst als %%BTRFS|btrfs%% oder %%ZFS|zfs%% neu formatieren. Deshalb ist %%BTRFS|btrfs%% das Standard-Dateisystem für %%cache pools|cache-pool%%, selbst wenn nur ein Gerät verwendet wird. Sowohl %%BTRFS|btrfs%% als auch %%ZFS|zfs%% unterstützen das spätere Hinzufügen von Geräten, um spiegelnde Pools zu erstellen. Für weitere Informationen zur Dateisystem-Auswahl und -Kompatibilität siehe [Dateisysteme](../file-systems.mdx).
:::

Der Einzelgerätemodus ist unkompliziert. Sie können das Gerät leicht hinzufügen oder entfernen, haben jedoch keine Optionen für erweiterte Funktionen wie Redundanz oder Erweiterung.
