# Unraid aktualisieren

Das Aktualisieren von Unraid OS stellt sicher, dass Sie die neuesten Funktionen, Sicherheitsupdates und Hardwareunterstützung erhalten. Diese Seite beschreibt den standardmäßigen Aktualisierungsprozess sowie Problemlösungs-Tipps und manuelle Upgrade-Optionen.

:::note\[Prerequisites]

Bevor Sie mit der Aktualisierung beginnen, sichern Sie unbedingt eine vollständige Sicherung Ihres USB-Flash-Geräts. Weitere Einzelheiten finden Sie unter [Sicherung Ihres Flash-Geräts](../maintain-and-update/changing-the-flash-device.mdx#backing-up-your-flash-device).

:::

:::tip[Best Praktiken vor dem Upgrade]

1. **Erstellen Sie ein Backup** Ihres Flash-Laufwerks und aller wichtigen Daten.
2. **Lesen Sie die Versionshinweise** für die Version, die Sie installieren möchten.
3. **Aktualisieren Sie alle Plugins** auf die neuesten Versionen.
4. **Stoppen Sie optional das Array**, bevor Sie fortfahren.
5. **Aktualisieren Sie Unraid OS** über die Seite ***Werkzeuge → Update OS***.
6. **Starten Sie** Ihren Server neu, um das Upgrade abzuschließen.

:::

:::caution[ZFS Pool-Upgrade-Warnungen]

Beim Upgrade auf Unraid 7.x können während des Bootvorgangs oder im WebGUI Warnungen zu [%%ZFS|zfs%%](../../advanced-configurations/optimize-storage/zfs-storage.mdx) Pool-Feature-Upgrades angezeigt werden. Diese Warnungen sind kein Anzeichen für ein Problem; sie weisen lediglich darauf hin, dass Ihr %%ZFS|zfs%%-Pool Funktionen einer älteren Version von ZFS verwendet.

Das Upgrade Ihres Pools ist nicht dringend erforderlich, aber wenn Sie sich entscheiden, es durchzuführen, sollten Sie beachten, dass es möglicherweise nicht mit früheren Versionen von Unraid funktioniert. Dies bedeutet, dass Sie nach dem Upgrade möglicherweise nicht zu einer vorherigen Unraid-Version zurückkehren können.

Denken Sie wie immer daran, Ihre Daten zu sichern, bevor Sie Ihre %%ZFS|zfs%%-Pools aktualisieren.

:::

<details>
  <summary><strong>VMs zeigen "Kann Schnittstellen-MTU nicht abrufen" oder Netzwerkfehler</strong> - Klicken zum Expandieren/Reduzieren</summary>

  Wenn Sie auf Fehler wie *"Layer aus Manifest stimmen nicht mit der Bildkonfiguration überein"* stoßen, müssen Sie möglicherweise Ihre Docker-Image-Datei neu erstellen. So geht's:

  1. Gehen Sie zu ***Einstellungen → Docker*** und stoppen Sie den Docker-Dienst.
  2. Geben Sie dann `cd efi/boot` ein.
  3. Schließlich geben Sie `bootx64.efi` ein.
  4. Navigieren Sie zur **Apps**-Registerkarte, **Vorherige Apps** und markieren Sie, was Sie erneut installieren möchten, und klicken Sie auf **„xx Anwendungen installieren“**.
</details>

<details>
  <summary><strong>VNC-Zugriff auf VMs funktioniert nicht oder ist langsam</strong> - Klicken zum Expandieren/Reduzieren</summary>

  Um dieses Problem zu lösen, bearbeiten Sie die %%VM|vm%% im %%WebGUI|web-gui%% und klicken Sie auf **Anwenden**, ohne Änderungen vorzunehmen. Diese Aktion aktualisiert den Maschinentyp auf die neueste unterstützte Version.

  1. Gehen Sie zum **VMs**-Tab und bearbeiten Sie jede %%VM|vm%% (stellen Sie sicher, dass die **Erweiterte Ansicht** aktiviert ist).
  2. Setzen Sie die Netzwerkbrücke auf `br0` und klicken Sie auf **Anwenden**.
  3. Navigieren Sie zu ***Einstellungen → VM Manager*** (in **Erweiterter Ansicht**) und setzen Sie die Standardbrücke auf `br0`.
</details>

<details>
  <summary><strong>VM wird nicht booten (EFI-Shell erscheint)</strong> - Klicken zum Expandieren/Reduzieren</summary>

  Wenn Sie %%OVMF|ovmf%%-basierte %%VMs|vm%% in älteren Unraid-Versionen erstellt haben, könnten Sie auf eine EFI-Shell stoßen. Sie können die %%VM|vm%% starten, indem Sie die folgenden Befehle eingeben:

  1. Bearbeiten Sie die %%VM|vm%% über das **VMs**-Tab (wählen Sie **Erweiterte Ansicht** aus).
  2. Stellen Sie den **%%VNC|vnc-session%%-Video-Treiber** auf **QXL** (empfohlen) ein. Versuchen Sie **Cirrus** oder **vmvga**, wenn Sie mit QXL wenig Erfolg haben.
  3. Klicken Sie auf **Anwenden**, um die Änderungen zu speichern.
</details>

<details>
  <summary><strong>VMs zeigen "Kann Schnittstellen-MTU nicht abrufen" oder Netzwerkfehler</strong> - Klicken zum Expandieren/Reduzieren</summary>

  Wenn Sie %%OVMF|ovmf%%-basierte %%VMs|vm%% in älteren Unraid-Versionen erstellt haben, könnten Sie auf eine EFI-Shell stoßen. Sie können die %%VM|vm%% starten, indem Sie die folgenden Befehle eingeben:

  1. Geben Sie `fs0:` ein.
  2. Geben Sie dann `cd efi/boot` ein.
  3. Schließlich geben Sie `bootx64.efi` ein.

  Wenn `fs0:` nicht funktioniert, können Sie stattdessen `fs1:` versuchen. Wenn weiterhin Probleme auftreten, besuchen Sie bitte das [Unraid-Forum](https://forums.unraid.net/) für Unterstützung.
</details>

## Manuelles Downgrade

Manuelle Downgrades sind selten notwendig, können aber erforderlich sein, wenn Sie keinen Zugriff auf das %%WebGUI|web-gui%% haben oder zu einer vorherigen Version zurückkehren müssen. Bevor Sie fortfahren, ist es wichtig, Ihr USB-Flash-Laufwerk zu sichern. Details dazu finden Sie unter [Sichern des Flash-Laufwerks](../maintain-and-update/changing-the-flash-device.mdx#backing-up-your-flash-device).

:::info[What tritt während des Downgrade-Prozesses auf]

Beim Verwenden des Downgrade-OS-Tools (WebGUI) erledigt das Tool automatisch Folgendes:

1. **Systemvalidierung**: Es wird überprüft, ob das Downgrade sicher durchgeführt werden kann.
2. **Dateisicherung**: Eine Sicherung Ihrer aktuellen Systemdateien wird erstellt.
3. **Versionsüberprüfung**: Das Tool stellt sicher, dass die Dateien für die Zielversion gültig und vollständig sind.
4. **Dateiaustausch**: Es ersetzt die aktuellen OS-Dateien durch die Dateien aus der vorherigen Version.
5. **Konfigurationsbewahrung**: Ihre aktuellen Einstellungen bleiben erhalten.
6. **Systemvorbereitung**: Das System wird für die vorherige Version vorbereitet.
7. **Neustartplanung**: Ein Neustart wird geplant, um das Downgrade abzuschließen.

:::

#### Post-Downgrade-Verifizierung

Nach Abschluss des Downgrades und dem Neustart Ihres Servers:

1. **Version prüfen**: Überprüfen Sie, ob Sie die erwartete Version von Unraid ausführen.
2. **Test core functionality**: Ensure that the %%array|array%%, Docker, and VM services are working properly.
3. **Plugins verifizieren**: Überprüfen Sie, ob Ihre installierten Plugins kompatibel und funktionsfähig sind.
4. **Docker-Container testen**: Starten Sie Ihre Docker-Container und bestätigen Sie, dass sie korrekt funktionieren.
5. **VMs überprüfen**: Wenn Sie über virtuelle Maschinen (VMs) verfügen, stellen Sie sicher, dass sie starten und ordnungsgemäß arbeiten. Siehe [VM-Einrichtung](../../using-unraid-to/create-virtual-machines/vm-setup.mdx) für Konfigurationsrichtlinien.
6. **Systemprotokolle überwachen**: Behalten Sie die Systemprotokolle im Auge, um Fehlermeldungen zu erkennen. Verwenden Sie [Diagnosetools](../../troubleshooting/diagnostics/capture-diagnostics-and-logs.mdx), um detaillierte Systeminformationen zu erfassen.

#### Problembehebung bei Downgrade-Problemen

<details>
  <summary><strong>Downgrade-Option nicht verfügbar</strong> - Zum Erweitern/Zusammenklappen klicken</summary>

  Wenn Sie keine Option für das Downgrade sehen:

  - **Systemhistorie überprüfen**: Stellen Sie sicher, dass Sie zuvor von einer kompatiblen Version aktualisiert haben.
  - **Dateiintegrität verifizieren**: Die Dateien der vorherigen Version könnten beschädigt oder fehlen.
  - **Kompatibilität überprüfen**: Die frühere Version ist möglicherweise nicht mit Ihrer aktuellen Hardware kompatibel.
  - **Manuelle Methode**: Befolgen Sie bei Bedarf die manuellen Downgrade-Methoden auf dieser Seite.
</details>

<details>
  <summary><strong>Downgrade schlägt während des Prozesses fehl</strong> - Zum Erweitern/Zusammenklappen klicken</summary>

  Wenn der Downgrade-Prozess nicht wie geplant verläuft:

  1. **Keine Panik**: Ihr System sollte weiterhin mit der aktuellen Version funktionieren.
  2. **Protokolle überprüfen**: Werfen Sie einen Blick auf die Systemprotokolle, um spezifische Fehlermeldungen zu finden. Verwenden Sie [Diagnosetools](../../troubleshooting/diagnostics/capture-diagnostics-and-logs.mdx) für eine umfassende Protokollanalyse.
  3. **Dateien verifizieren**: Stellen Sie sicher, dass alle erforderlichen Dateien für die Zielversion verfügbar sind.
  4. **Probieren Sie die manuelle Methode**: Wenn das Tool nicht funktioniert, ziehen Sie die Verwendung der manuellen Downgrade-Methoden in Betracht.
  5. **Kontaktieren Sie den Support**: Wenn die Probleme weiterhin bestehen, wenden Sie sich mit Ihrer Diagnose an den Unraid-Support.
</details>

<details>
  <summary><strong>System startet nach Downgrade nicht</strong> - Zum Erweitern/Zusammenklappen klicken</summary>

  Wenn Ihr System nach dem Downgrade nicht startet:

  1. **Vom USB starten**: Verwenden Sie einen anderen Computer, um auf Ihr USB-Flash-Laufwerk zuzugreifen. Siehe [Startfehlerbehebung](../../troubleshooting/common-issues/boot-and-startup-failures.mdx) für detaillierte Wiederherstellungsschritte.
  2. **Von Sicherung wiederherstellen**: Kopieren Sie Ihre Sicherungsdateien zurück auf das USB-Gerät.
  3. **Dateiintegrität überprüfen**: Stellen Sie sicher, dass alle Systemdateien vorhanden und nicht beschädigt sind.
  4. **Versuchen Sie den sicheren Modus**: Booten Sie in den [Sicheren Modus](../../using-unraid-to/customize-your-experience/plugins.mdx#troubleshooting-with-safe-mode) für weitere Fehlerbehebungen.
  5. **Manuelle Wiederherstellung**: Falls erforderlich, nutzen Sie die manuellen Wiederherstellungsmethoden.
</details>

<details>
  <summary><strong>Plugins oder Docker-Container funktionieren nicht</strong> - Zum Erweitern/Zusammenklappen klicken</summary>

  Nach dem Downgrade funktionieren einige Plugins oder Container möglicherweise nicht ordnungsgemäß:

  - **Plugins aktualisieren**: Suchen Sie nach Plugin-Updates, die mit der heruntergestuften Version funktionieren. Überprüfen Sie [Community-Anwendungen](../../using-unraid-to/run-docker-containers/community-applications.mdx) auf kompatible Versionen.
  - **Docker neu erstellen**: Löschen und erstellen Sie Ihre Docker-Image-Datei neu. Siehe [Docker-Fehlerbehebung](../../troubleshooting/common-issues/docker-troubleshooting.mdx#re-creating-the-docker-image-file) für detaillierte Schritte.
    - **Abhängigkeiten überprüfen**: Stellen Sie sicher, dass alle erforderlichen Abhängigkeiten verfügbar sind.
    - **Plugin-Kompatibilität**: Einige Plugins unterstützen möglicherweise keine älteren Unraid-Versionen.
    - **UI/WebGUI abhängige Plugins**: Plugins, die von neueren WebGUI-Funktionen abhängen, funktionieren möglicherweise nicht richtig in älteren Versionen.
    - **Themen-Plugins**: Visuelle Anpassungs-Plugins können Aktualisierungen oder das Entfernen erfordern, wenn sie nicht kompatibel sind.
</details>
